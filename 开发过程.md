# 开发过程

## 目的

这个文档用于持续记录项目关键修改、对应功能影响、共通注意事项，以及后续对话需要快速掌握的上下文。

## 更新规则

- 关键修改包括：新增功能、重要交互调整、核心逻辑变更、重要 bug 修复、资源组织调整、构建或发布约束变更。
- 每次完成关键修改后，都要追加一条记录，至少写清楚改了什么、影响了什么功能、涉及哪些文件、还有什么约束。
- 新增共通注意事项时，先更新本文档，再同步检查 .github 下的 instructions 或 skill 是否需要一起更新。
- 不记录密钥、账号、令牌等敏感信息。

## 项目概览

- 这是一个以静态资源和前端脚本为主的项目，当前结构以 index.html、css、js、assets 为核心。
- 项目目标包含 GitHub Pages 发布，因此默认以纯静态站点兼容为前提，不引入服务端依赖。
- docs 目录用于沉淀需求和素材文档；.github 目录用于 Copilot 协作配置，不参与页面运行时逻辑。

## 共通注意事项

- 默认不要引入会破坏 GitHub Pages 部署前提的服务端方案、运行时依赖或无关构建污染。
- 每次新开对话时，先阅读这个文档，再开始理解和修改项目。
- 后续如果你补充新的共通点，就把它们追加到这里，作为长期约束来源。

- 为避免一次性输出过长导致写入失败或响应截断：在创建或写入多个文件前，先整理并列出要创建的文件框架与写入步骤，按分批次（分块）写入并逐步校验每一步结果。
- 为避免处理超大图片或请求过大：在批量处理图片前，先把图片素材上传或迁移到可通过 Web 访问的位置（并尽量生成适当分辨率的副本），然后按小批量（一张或少量一组）下载并逐张处理与核对，记录每批次结果后再继续下一批。

### 2026-04-04 (3) — 许愿环节关灯效果修复

- 背景：许愿环节的关灯效果依赖 canvas 内部整块压暗，导致仪式区出现明显黑框，蜡烛与蛋糕处在不同光照层里，观感割裂。
- 关键变动：将仪式暗场主逻辑收敛到页面级遮罩层，提升遮罩层级到 canvas 之上；移除 canvas 内整块黑色蒙版，改为更柔和的烛光受光层与全屏暗场渐变叠加，让蛋糕和蜡烛共享同一套灯光氛围。
- 影响功能：优化模块7 许愿环节的关灯、点蜡烛与许愿阶段视觉表现，不改变点蜡烛、吹蜡烛或模块跳转逻辑。
- 涉及文件：js/modules/ceremony.js、css/modules.css。
- 备注 / 后续：建议在移动端和桌面端各走一遍许愿流程，观察不同数量蜡烛下的暗场强度是否还需要继续微调。

### 2026-04-04 (4) — 色素选择与蜡笔笔触调整

- 背景：色素瓶底部文案与瓶身颜色对不上；许愿页暗场仍有无来源的中心打光；用户希望把涂鸦进一步做成蜡笔质感，并预留灯光/音乐按钮入口。
- 关键变动：移除奶油制作阶段色素瓶底部文字，只保留瓶身选择；把许愿页关灯后的环境光改为仅由点亮蜡烛产生，不再保留蛋糕中心的固定打光；新增灯光、音乐两个按钮作为后续挑战入口；增强蜡笔笔触的颗粒和叠涂纹理，并在调色板中加入黑色。
- 影响功能：优化模块3 选色 UI、模块4/5 涂鸦质感与颜色选择、模块7 许愿环节控件和关灯后的光照表现，不改变既有流程跳转。
- 涉及文件：index.html、css/modules.css、js/config.js、js/modules/creamMaking.js、js/modules/ceremony.js、js/utils.js。
- 备注 / 后续：可继续补充灯光/音乐按钮的真实玩法逻辑；若后续想再偏粉笔或油画风，可以继续微调蜡笔笔触参数。

### 2026-04-04 (5) — 手机端色素瓶改为点击上色

- 背景：手机端色素瓶沿用 Pointer 拖拽流程，实际操作中不易拖动到碗内，影响模块3 奶油上色体验。
- 关键变动：将色素瓶交互拆分为桌面端拖拽、触摸设备点击两套模式；手机端点击色素瓶即可直接触发倒色动画；同步把上色提示文案改为按设备类型显示，并给色素瓶补充 `touch-action: manipulation`。
- 影响功能：优化模块3 在手机端的上色操作，不影响桌面端原有拖拽流程，也不改变后续打发和抹面逻辑。
- 涉及文件：css/modules.css、js/modules/creamMaking.js。
- 备注 / 后续：建议真机走一遍模块3，确认不同手机浏览器下点击反馈和横向滚动色素栏的手感是否还需要继续细调。

### 2026-04-04 (6) — 手机端蜡烛拖拽坐标修复

- 背景：模块5 装饰阶段在手机端拖放蜡烛时，释放落点的坐标换算错误，导致蜡烛跳到画布左上区域。
- 关键变动：补强公共画布坐标工具，对 `touchend` 优先读取 `changedTouches`；同时修正装饰模块在拖放结束时的落点换算，直接使用当前触点对象计算 Canvas 坐标。
- 影响功能：修复模块5 手机端拖放蜡烛和其他装饰素材时的位置偏移问题，不影响桌面端鼠标拖拽逻辑。
- 涉及文件：js/modules/decorate.js、js/utils.js。
- 备注 / 后续：建议在手机端再验证一次“从素材栏拖到蛋糕上”和“拖动已放置蜡烛重新定位”这两条路径，确认都不再偏移。

### 2026-04-04 (7) — 全流程加入重头制作按钮

- 背景：为了应对中途误操作或流程卡住的情况，需要在任意制作阶段快速放弃当前草稿并重新开始，同时保留已存入相册的成品。
- 关键变动：在全局顶部操作区新增“重头制作”按钮；点击后弹出确认，清空当前 `zzhappy-state` 对应的蛋糕制作进度并刷新回开头；按钮默认在封面隐藏，在进入制作流程后全程可用。
- 影响功能：用户可以从模块2 之后任意阶段放弃当前未完成蛋糕并重新开始；相册数据因存储在独立的 IndexedDB，不受影响。
- 涉及文件：index.html、css/main.css、js/app.js。

### 2026-04-04 (8) — 仪式暗场、奶油流程与裱花笔触重做

- 背景：现有关灯效果带有模糊滤镜，蜡烛不够清晰；奶油制作顺序与期望不一致，且进入模块时有明显分段加载感；蛋糕绘制笔触也需要从蜡笔改成更像奶油裱花的效果。
- 关键变动：移除许愿页的灯光/音乐占位按钮，去掉暗场模糊并加深整体黑度，同时加强烛光局部高光；将奶油流程改为“打鸡蛋 → 打发奶油 → 滴入色素 → 抹面”，并对奶油制作模块增加并行预载和加载态，减少首屏分块出现的卡顿感；把原来的蜡笔笔触改为带立体高光和挤压纹理的奶油裱花笔触，同步更新界面文案。
- 影响功能：优化模块3 奶油制作的节奏与首屏体验，优化模块5/截图/许愿页里裱花与成品的视觉风格，并收紧模块7 的暗场表现。
- 涉及文件：index.html、css/modules.css、js/modules/ceremony.js、js/modules/creamMaking.js、js/modules/creamPaint.js、js/modules/decorate.js、js/utils.js。
- 备注 / 后续：建议重点在移动端实测三段流程：进入奶油制作页的加载感、打发后选色再抹面的节奏、以及裱花笔触在浅色和深色下的立体感是否还需要继续微调。

### 2026-04-04 (9) — 手机端吹蜡烛识别增强

- 背景：模块7 在桌面端可通过麦克风吹灭蜡烛，但手机端由于默认降噪、自动增益和固定阈值过高，吹气信号经常无法达到原先的识别条件。
- 关键变动：将麦克风申请改为优先请求单声道且尽量关闭回声消除、降噪和自动增益；把原先仅依赖频谱平均值的固定阈值检测，改为结合频域能量、低频能量、时域峰值和 RMS 的动态基线检测；补充吹气检测循环与音频资源的统一清理逻辑。
- 影响功能：提升模块7 在手机浏览器上的吹气识别成功率，同时保留点击火焰手动熄灭的兜底路径；桌面端麦克风吹气逻辑继续可用。
- 涉及文件：js/modules/ceremony.js、开发过程.md。
- 备注 / 后续：建议优先在 iPhone Safari 和 Android Chrome 真机验证一次，关注首次授权后直接吹气、环境噪声较大时的误触发率，以及拒绝麦克风权限后的点击兜底提示是否清晰。
