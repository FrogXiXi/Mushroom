# 开发过程

## 目的

这个文档用于持续记录项目关键修改、对应功能影响、共通注意事项，以及后续对话需要快速掌握的上下文。

## 更新规则

- 关键修改包括：新增功能、重要交互调整、核心逻辑变更、重要 bug 修复、资源组织调整、构建或发布约束变更。
- 每次完成关键修改后，都要追加一条记录，至少写清楚改了什么、影响了什么功能、涉及哪些文件、还有什么约束。
- 新增共通注意事项时，先更新本文档，再同步检查 .github 下的 instructions 或 skill 是否需要一起更新。
- 不记录密钥、账号、令牌等敏感信息。

## 项目概览

- 这是一个以静态资源和前端脚本为主的项目，当前结构以 index.html、css、js、assets 为核心。
- 项目目标包含 GitHub Pages 发布，因此默认以纯静态站点兼容为前提，不引入服务端依赖。
- docs 目录用于沉淀需求和素材文档；.github 目录用于 Copilot 协作配置，不参与页面运行时逻辑。

## 共通注意事项

- 默认不要引入会破坏 GitHub Pages 部署前提的服务端方案、运行时依赖或无关构建污染。
- 每次新开对话时，先阅读这个文档，再开始理解和修改项目。
- 后续如果你补充新的共通点，就把它们追加到这里，作为长期约束来源。

- 为避免一次性输出过长导致写入失败或响应截断：在创建或写入多个文件前，先整理并列出要创建的文件框架与写入步骤，按分批次（分块）写入并逐步校验每一步结果。
- 为避免处理超大图片或请求过大：在批量处理图片前，先把图片素材上传或迁移到可通过 Web 访问的位置（并尽量生成适当分辨率的副本），然后按小批量（一张或少量一组）下载并逐张处理与核对，记录每批次结果后再继续下一批。

- 关键变动：让奶油模块的色素瓶在打蛋/打发阶段就常驻底部，并通过禁用态提示“先打发、再滴色”；打发完成后新增右下角“完成打发奶油，去滴入色素”按钮，滴色完成后新增“完成滴入色素，去抹面”按钮；吹蜡烛模块改为点亮全部蜡烛后显示“许完愿了，去吹蜡烛”，吹灭全部蜡烛后显示“完成吹蜡烛，去切蛋糕”，不再自动跳转；背景增加更多 doodle 素材位，并重新平衡手机端与桌面端的尺寸；同步放大手机端抹刀、手机端和桌面端搅拌器尺寸。
- 影响功能：优化模块3 奶油制作与模块7 许愿仪式的节奏停顿感，减少突然切换；提升全站背景装饰层次，以及制作工具在不同设备上的可见性与可操作性。
- 涉及文件：index.html、css/modules.css、js/modules/creamMaking.js、js/modules/ceremony.js、开发过程.md。
- 备注 / 后续：建议在手机端真机重点确认三个节点的手感：打发完成后按钮是否遮挡视线、滴色完成后的停顿是否自然、吹灭蜡烛后再切蛋糕的节奏是否还需要更长的留白。

### 2026-05-30 (2) — 合并打发与滴色界面 & 修复仪式按钮定位

- 背景：奶油制作模块打发完成后弹出"完成打发奶油，去滴入色素"按钮是多余的中间步骤，用户希望打发和滴色在同一个界面无缝衔接；另外仪式模块的"去吹蜡烛"和"去切蛋糕"按钮在页面上不可见，存在定位 bug。
- 关键变动：
  - **js/modules/creamMaking.js**：`_onWhipDone()` 不再设置 `step='whip-ready'` 并显示中间按钮，改为直接切换到 `step='color'`、显示色素瓶选择器并启用交互，toast 提示"奶油打发好了，选个颜色给奶油上色吧"。同步清理 `_setStageLayout()` 中已不存在的 `whip-ready` 分支。
  - **css/modules.css**：移除 `.ceremony-actions` 上的 `position: relative`（该属性导致内部 `.btn-next` 的 `position: absolute; bottom: 5vh; right: 5vw` 相对于一个高度塌缩的 flex 容器定位，按钮跑到不可见区域）；同步移除无效的 `z-index: 24`（没有 position 时 z-index 不生效）。
- 影响功能：模块3 奶油制作流程简化为"打蛋 → 打发 → 选色（直接衔接）→ 滴色动画 → 抹面"；模块7 仪式阶段的"许完愿了，去吹蜡烛"和"完成吹蜡烛，去切蛋糕"按钮恢复正常定位，桌面端和移动端均可点击。
- 涉及文件：js/modules/creamMaking.js、css/modules.css。
- 备注 / 后续：已通过 Playwright 桌面端截图验证两项修复均生效；代码审查确认无残留引用、无内存泄漏、无竞态风险。
