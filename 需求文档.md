# 需求文档：Hello Kitty蜡笔手绘风生日互动网站
## 文档版本：V1.0
## 核心交付约束【强制遵守，不可修改】
最终交付**单文件内嵌式HTML**，所有CSS/JS全部内嵌在HTML文件内，无任何外部本地文件依赖；**纯前端实现，无后端、无数据库、无服务端接口、无数据上报**，打开HTML文件即可完整运行全功能。

## 一、项目概述
### 1.1 项目目标
复刻参考图的蜡笔手绘软萌风格，开发一个以「麦克风吹蜡烛」为核心玩法的沉浸式生日互动网站，实现完整的生日仪式闭环，零学习成本，打开即可使用，兼容PC端与移动端全设备。
### 1.2 参考风格基准
严格对齐参考图的视觉风格：全页面统一儿童蜡笔手绘质感，所有元素带不规则手绘描边、色彩柔和晕染，无尖锐直角、无浏览器原生控件，主色调为樱花粉+奶白色+暖黄色，深棕色蜡笔描边，核心IP为Hello Kitty及蜡笔风小动物伙伴。

## 二、功能模块需求
### 2.1 P0级核心必选功能【必须100%实现，不可遗漏】
#### 2.1.1 核心王牌功能：麦克风吹蜡烛
##### 技术强制要求
- 必须使用 `Web Audio API + AnalyserNode` 实现，禁止仅用音量阈值判定
- 采用**「80-300Hz低频白噪音频谱识别+持续时长≥300ms+动态环境阈值」三维判定模型**，仅对吹气的低频特征响应，彻底避免说话、环境噪音误触
- 初始化麦克风时，先采集2s环境底噪自动校准判定阈值，手机端自动降低阈值、PC端自动抬高阈值，保证双端体验一致
##### 交互流程
1.  解锁时机：蛋糕上所有蜡烛全部点亮后，才解锁麦克风检测功能，弹出蜡笔风提示弹窗：「✨ 愿望默念好了吗？对着麦克风用力吹一口气，熄灭蜡烛吧～」
2.  实时反馈：页面底部常驻蜡笔风「气流可视化进度条」，实时显示吹气强度
    - 吹气力度不达标：对应蜡烛火焰剧烈晃动+烛杆抖动，进度条变红，弹出提示「再用力一点呀～」，同步播放轻量「呼」风声效
    - 吹气力度达标且持续≥300ms：对应蜡烛火焰收缩→冒白烟→熄灭，同步播放「噗」的灭烛音效，页面亮度轻微下调
3.  分批吹灭规则：支持单根/多根分批吹灭，离画面中心越近的蜡烛优先熄灭，无需一次性吹完
4.  全灭触发：最后一根蜡烛熄灭后，先定格0.5s黑屏（仅保留蜡烛白烟动画），再触发全屏狂欢特效
##### 容错兜底方案【强制实现】
- 用户拒绝麦克风权限/设备无麦克风：**无感自动切换为鼠标长按吹蜡烛备用模式**，长按蜡烛位置，气流进度条增长，进度条满即可熄灭蜡烛，不阻断核心流程
- 麦克风异常/断开：自动弹出提示，一键切换备用模式
- 蜡烛未全部点亮前，麦克风检测完全关闭；单根蜡烛熄灭后设置1s冷却时间，避免连续误触
- 吹气时，同步触发页面元素风吹动效：Hello Kitty蝴蝶结、小动物耳朵、蛋糕裱花、背景彩旗轻微晃动

#### 2.1.2 蜡烛基础交互系统
1.  点亮仪式：必须按从中间到两边的顺序，鼠标/触屏点击蜡烛逐根点亮，每点亮一根同步播放「咔哒」点火音效+火焰摇曳动画，全点亮后触发集体欢呼音效
2.  重置功能：蜡烛全灭后，页面常驻蜡笔风「重新点亮蜡烛」按钮，可无限次重复体验核心流程
3.  数字蜡烛：支持选择年龄，蛋糕自动生成对应数字蜡烛，替换普通蜡烛，数字蜡烛优先熄灭

#### 2.1.3 生日专属定制系统
1.  入口：进入页面首先展示全屏蜡笔风定制面板，必填项仅「寿星姓名」，年龄、头像、主题色为可选项，可跳过
2.  定制项实现要求：
    - 寿星姓名：输入后自动生成蜡笔艺术字，固定在页面顶部「祝XXX 生日快乐」
    - 年龄选择：选择后自动生成对应数字蜡烛
    - 头像上传：支持上传图片，通过Canvas纯前端实现蜡笔风描边效果，生成的头像放在蛋糕C位与Hello Kitty同框
    - 主题切换：支持粉色Hello Kitty/蓝色布丁狗/黄色玉桂狗，一键切换全页面配色、描边、装饰元素
3.  纯前端分享实现：所有定制内容自动编码到URL哈希参数中，分享链接打开后自动还原定制内容，无后端依赖

#### 2.1.4 蜡笔小动物互动系统
1.  布局：蛋糕周围环形排布Hello Kitty、小熊、小狗、小猫等蜡笔风小动物，与参考图风格完全统一
2.  流程绑定交互：
    - 定制环节：点击小动物触发蹦跳动画+专属音效，同步举牌提示操作
    - 点亮蜡烛：每点亮一根，对应方向的小动物挥手欢呼蹦跳
    - 吹蜡烛环节：小动物保持捂嘴期待的静态动画，吹气时同步随风晃动
    - 全灭狂欢：所有小动物围圈跳舞，撒爱心/糖果粒子，随机举牌「生日快乐」「愿望成真」「永远可爱」
3.  常驻交互：任意时刻点击小动物，都触发专属音效+动画，无冷却时间

#### 2.1.5 沉浸式氛围控制系统
1.  光线自动递进：
    - 入场页：明亮蜡笔风暖光
    - 点亮蜡烛：每点亮一根，页面环境光暗一分，蜡烛光影效果增强
    - 全点亮后：切换为暗调柔光，仅保留蜡烛动态光影，营造许愿氛围
    - 全灭后：0.5s黑屏→瞬间亮屏，切换为明亮狂欢模式
2.  音效自动适配：
    - 全程低音量循环蜡笔风钢琴BGM，可一键暂停
    - 吹蜡烛环节，BGM自动静音，仅保留吹气/灭烛音效
    - 蜡烛全灭后，自动播放《生日快乐》纯音乐，支持暂停/切歌
    - 所有音效使用CC0无版权素材，通过公开CDN引入
3.  全局特效：
    - 鼠标/触屏移动：拖尾蜡笔笔触/爱心/星星，点击触发迷你烟花特效
    - 狂欢环节：全屏飘落彩带、蝴蝶结、小蛋糕、糖果粒子，支持一键关闭
4.  性能降级：检测到设备性能不足/帧率过低时，自动关闭粒子特效、复杂动画，仅保留核心吹蜡烛功能，保证流程不卡顿

#### 2.1.6 电子贺卡生成功能
- 集成html2canvas（通过公开稳定CDN引入），纯前端实现DOM转图片
- 蜡烛全灭后，可一键生成包含蛋糕、寿星姓名、头像、祝福内容的高清PNG贺卡，支持保存到本地
- 所有按钮、输入框均为蜡笔风设计，禁止使用浏览器原生控件

### 2.2 P1级优先实现功能
#### 2.2.1 左侧导航栏功能（对齐参考图）
1.  **CAKE DECORATING（蛋糕DIY）**：支持拖动蝴蝶结、樱桃、糖果、草莓等蜡笔风装饰，自由摆放在蛋糕上，装饰自动贴合蛋糕弧度；支持切换奶油草莓/巧克力/芒果慕斯蛋糕样式
2.  **GUESTBOOK（祝福留言墙）**：纯前端实现，输入祝福+昵称，自动生成蜡笔便签展示在侧边栏；祝福内容自动编码到URL哈希参数中，分享链接后好友可查看/新增祝福
3.  **PARTY INVITE（派对邀请）**：一键复制带定制参数的页面链接，提示用户分享给好友
4.  **GAMES & FUN**：折叠收纳隐藏彩蛋功能入口
5.  **GIFT SHOP**：展示Hello Kitty主题虚拟装饰素材，点击可应用到页面

#### 2.2.2 许愿与手写祝福功能
1.  私密许愿弹窗：蜡烛全灭后自动弹出，输入的愿望仅存储在localStorage，本地可见，刷新不丢失
2.  蜡笔手写祝福板：Canvas实现手写画布，自带蜡笔笔触效果，支持选择颜色、笔触粗细

#### 2.2.3 隐藏彩蛋功能
1.  连续点击Hello Kitty 5次：触发专属舞蹈动画+音效
2.  双击蛋糕：掉落满屏糖果，小动物触发抢糖果动画
3.  长按屏幕空白处：触发全屏鼓掌特效，小动物同步鼓掌
4.  姓名输入框输入「永远可爱」：解锁全限定装饰素材

### 2.3 P2级可选实现功能
1.  背景场景切换：支持粉色公主房/星空派对/草地野餐3种蜡笔风背景，一键切换
2.  夜晚模式：一键切换暗黑背景，增强蜡烛光影氛围感
3.  横屏适配：手机横屏时自动切换横版布局，核心元素始终居中

### 2.4 ❌ 禁止实现的功能【绝对不可添加】
1.  任何需要后端/服务器/数据库支持的功能，包括但不限于实时多人互动、账号登录系统、数据永久云存储
2.  短链接生成、访问量统计、用户数据上报等需要第三方接口的功能
3.  任何广告、跳转链接、第三方引流内容
4.  不符合蜡笔手绘风格的原生浏览器控件、扁平化元素

## 三、页面体验动线【严格遵循，不可打乱】
1.  暖场入场页：蜡笔Hello Kitty蹦跳加载动画，加载完成自动跳转，无多余操作
2.  专属定制页：全屏蜡笔风定制面板，完成定制后点击「开启生日派对」进入主页面
3.  仪式点亮页：居中展示蛋糕+小动物，仅开放蜡烛点亮操作，其余功能折叠锁定
4.  许愿吹烛页：蜡烛全亮后解锁麦克风，引导吹蜡烛，完成核心仪式
5.  全场狂欢页：蜡烛全灭后触发狂欢动画+生日歌，自动弹出许愿框
6.  祝福留存页：开放所有拓展功能，引导用户写祝福、生成贺卡、分享链接

## 四、技术实现强制要求
1.  代码规范：模块化拆分，关键逻辑添加清晰注释，便于后续修改调整
2.  兼容性要求：
    - 兼容Chrome、Edge、Safari等主流PC浏览器
    - 兼容iOS 13+、Android 10+移动端浏览器
    - 针对iOS Safari强制要求：`AudioContext` 必须在用户点击交互后初始化，避免麦克风功能失效
3.  素材要求：所有视觉元素优先使用内联SVG矢量图，保证加载速度与缩放无损；所有素材、音效均为可免费商用的CC0协议内容
4.  隐私要求：无任何用户数据收集、上报行为，所有数据仅存储在用户本地localStorage与URL哈希中
5.  响应式适配：使用CSS媒体查询+弹性布局，实现从手机到PC全尺寸适配，触屏/鼠标操作全支持
