## 一、项目概述
### 1.1 项目背景
打造一款轻量化、高仪式感的线上生日蛋糕DIY互动网站，用户可完成从蛋糕制作、装饰、生日仪式、切割分享的全流程闭环，全程操作极简，氛围温暖治愈，核心突出「春尽夏生」的专属生日主题。全站视觉统一采用蜡笔画风，呈现纸面纹理、蜡质笔触、儿童绘本感与手账式生日仪式氛围。
### 1.2 核心目标
1. 1:1还原真实烘焙操作细节，实现沉浸式蛋糕DIY体验；
2. 完整落地生日仪式全流程，包含点蜡、许愿、吹蜡、专属祝福、切蛋糕的完整环节；
3. 核心功能无bug：自由切割蛋糕时，用户自定义涂鸦、装饰同步跟随蛋糕块切开，无错位、无脱节；
4. 全终端自适应，无需注册登录，打开即可使用，无广告、无付费环节。
### 1.3 目标用户
有生日仪式感需求的全年龄段个人用户，核心适配手机端触屏操作，兼容电脑端鼠标操作。
### 1.4 终端适配要求
- 优先适配：手机端（iOS/Android 主流机型，375px-430px 宽度）
- 兼容适配：平板端、电脑端（1024px-1920px 宽度）
- 自适应规则：所有交互区域、按钮、字体随屏幕宽度等比例缩放，核心操作区始终居中，无内容溢出、无按钮错位。

## 二、整体技术规范（开发必看）
### 2.1 推荐技术栈
- 前端框架：Vue3/React 或原生HTML+CSS+JavaScript（轻量化优先）
- 核心交互实现：Canvas 2D（必须，用于涂鸦、蛋糕切割、图层同步）
- 动画实现：CSS3 过渡动画 + Canvas 帧动画，保证流畅度
- 适配方案：rem/vw 自适应布局，禁止固定px尺寸（除边框、圆角外）

### 2.1.1 整体视觉风格规范（新增，必须遵守）
1. 全站统一采用蜡笔画风，不使用写实照片风、3D 材质风、扁平商务插画风；
2. 页面整体视觉需具备轻纸张纹理、蜡笔颗粒填色、轻微不规则描边三项特征，避免过于光滑、工业化的图形边缘；
3. 所有蛋糕、道具、装饰、图标、截图模板、提示卡片、按钮视觉均需统一为同一套蜡笔手绘语言，禁止出现单独风格跳脱的通用图标或写实素材；
4. 页面基础底色仍以暖奶黄系为主，但允许叠加极淡纸面纹理或蜡笔扫色，不得出现高对比渐变、金属高光、玻璃拟态效果；
5. 所有阴影、高光表现统一改为低对比的蜡笔扫色或奶油色涂抹块，不使用强真实投影、镜面高光；
6. 图标、边框、标签、相框可采用手绘描边、涂鸦角标、贴纸、胶带等蜡笔绘本元素增强整体感。

### 2.2 核心图层架构（必须严格遵守，直接影响切蛋糕功能实现）
所有蛋糕相关内容采用**四层绑定复合画布结构**，四层共用同一套相对坐标系统（以蛋糕中心为原点，蛋糕外轮廓为边界），禁止使用屏幕绝对坐标，确保缩放、移动、切割时所有图层完全同步：
| 图层顺序（从上到下） | 图层名称 | 存储内容 | 绑定规则 |
|------------------------|----------|----------|----------|
| 1（顶层） | 装饰元素层 | 用户拖拽摆放的蜡烛、水果、插件、插牌等装饰元素，存储每个元素的相对坐标、尺寸、旋转角度 | 与蛋糕本体完全绑定，元素坐标为相对于蛋糕中心的相对坐标，蛋糕移动/切割时，元素跟随对应蛋糕块同步变化 |
| 2 | 用户涂鸦层 | 用户抹奶油时绘制的所有涂鸦路径，存储每一笔的路径坐标、颜色、粗细 | 与蛋糕奶油层完全贴合，路径坐标为相对于蛋糕外轮廓的相对坐标，蛋糕切割时，涂鸦路径沿切割线同步裁剪断开 |
| 3 | 奶油底色层 | 蛋糕的奶油底色、抹面效果，存储奶油颜色、纹理 | 与蛋糕胚层完全绑定，为蛋糕的基础视觉层 |
| 4（底层） | 蛋糕胚层 | 单层/双层蛋糕胚的基础轮廓、底色 | 整个蛋糕的基础载体，所有上层图层均以该层的轮廓为边界 |

### 2.3 通用交互规范
1. 所有可交互元素（按钮、蛋糕刀、搅拌器等）必须有hover/点击反馈：触屏端点击时有缩放效果（0.95倍缩放，100ms过渡），电脑端hover时有亮度提升、指针变为手型；
2. 所有动画过渡时长统一规范：基础淡入淡出300ms，仪式感动画500-800ms，无生硬跳转、无闪屏；
3. 所有音效统一为轻柔治愈风格，音量默认30%，无刺耳音效，用户可手动关闭音效；
4. 所有操作无强制倒计时，用户可自主控制流程节奏，无卡顿、无崩溃。

### 2.4 蜡笔画风实现约束（开发/UI 联动规范）
1. 所有 PNG 素材需保留足够安全边距，避免蜡笔颗粒边缘在缩放、旋转、切割后被裁掉；
2. Canvas 中用于奶油、涂鸦、纸面质感的纹理应优先采用独立叠加素材或程序纹理，不建议把所有纸纹直接烘焙进主体素材，以便后续统一调整；
3. 蛋糕胚、奶油、装饰元素的视觉重心应尽量居中，方便后续拖拽、旋转、切割时维持自然观感；
4. 所有工具类图标中，“画笔”统一替换为蜡笔造型；“蛋糕刀”需弱化锋利感，表现为圆润手绘刀具；
5. 所有 UI 图标不直接使用通用系统图标，需二次绘制为蜡笔小图标或手绘描边图标。

## 三、分模块详细开发需求（按流程顺序）
### 模块1：首页封面模块
#### 1.1 功能描述
网站入口页，展示主题内容，引导用户进入制作流程，无任何冗余元素。
#### 1.2 视觉实现要求（严格遵守）
1. 背景：以**暖奶黄底色**为主，主色值参考：#FFF3D4；允许叠加极淡纸张纹理或蜡笔扫色肌理，但不得出现明显渐变、摄影质感或杂乱底纹；
2. 文字内容（分层展示）：
   - 主标题：书法大字**「春尽夏生」**，字体为手写书法体，色值#8B5A2B（深奶咖色），字号占屏幕宽度的40%，居中放置，占据页面核心视觉位置；
   - 副标题：主标题正下方，**To 【寿星名字】 · Happy Birthday · 202X.05.30**，字体为圆润无衬线圆体，色值#FFFFFF（淡奶油白），字号为屏幕宽度的4%，行高1.5，居中对齐；
3. 按钮：页面底部居中，圆角矩形按钮，尺寸为屏幕宽度的60%，高度48px，圆角24px，底色#8B5A2B，按钮文字**「点击进入」**，色值#FFF3D4，字体为粗圆体，字号16px；按钮边缘建议加入轻微手绘描边或蜡笔涂抹边缘，不使用标准电商按钮风格；
4. 页面点缀：允许在角落或标题周边加入少量蜡笔小星星、小花、小爱心、胶带贴纸等手账式点缀，但数量必须克制，不得遮挡主标题与按钮。
#### 1.3 交互&动画实现要求
1. 分层淡入动画（必须严格按顺序执行）：
   - 0ms：页面加载完成，背景层开始淡入，300ms完成完全显示；
   - 300ms：主标题「春尽夏生」开始淡入，同时带从下往上10px的位移动画，800ms完成完全显示；
   - 1100ms：副标题开始淡入，500ms完成完全显示；
   - 1600ms：「点击进入」按钮开始淡入，300ms完成完全显示，同时启动呼吸动效；
2. 按钮呼吸动效：循环执行，每2秒一个周期，缩放比例1.0→1.05→1.0，过渡平滑；
3. 点击按钮：点击后，整个页面执行淡出动画（500ms），同时跳转至蛋糕胚选择模块，无刷新、无白屏。
#### 1.4 边界规则
- 禁止添加任何广告、弹窗、导航栏、多余按钮；
- 文字内容禁止换行、溢出，小屏手机端自动缩放适配；
- 【寿星名字】【202X.05.30】为固定占位符，支持后续直接修改，无需用户手动输入。

### 模块2：蛋糕胚选择模块
#### 2.1 功能描述
用户选择蛋糕胚类型，为后续制作做基础准备。
#### 2.2 视觉实现要求
1. 背景：暖奶黄底色，与封面统一，可保留极淡纸张纹理；
2. 蛋糕胚选项：页面居中并列展示两个选项，每个选项包含蛋糕胚预览图+下方文字标注：
   - 选项1：单层圆形蛋糕胚，奶白色，预览图尺寸一致，蛋糕表面带蜡笔排线或颗粒质感；
   - 选项2：双层圆形蛋糕胚，底层大、上层小（比例3:2），奶白色，符合真实蛋糕比例，同时保持蜡笔手绘边缘；
3. 选中状态：选中的蛋糕胚选项，外围加2px的#8B5A2B实线边框，带轻微放大效果（1.05倍）；边框可处理为手绘感描边，不使用过于机械的标准直线框。
#### 2.3 交互实现要求
1. 点击任意蛋糕胚选项，立即切换选中状态，带300ms淡入淡出过渡动画；
2. 无需额外确认按钮，选中后，页面右下角出现「下一步」按钮，点击后跳转至奶油制作模块；
3. 默认选中「单层蛋糕胚」，用户可随时切换。
#### 2.4 边界规则
- 蛋糕胚比例固定，用户无需手动调整大小、比例；
- 切换无卡顿，预览图无变形。

### 模块3：奶油制作模块
#### 3.1 功能描述
还原真实烘焙流程：打鸡蛋→选奶油颜色→滴色素→打发奶油，全流程可交互。
#### 3.2 分步骤开发细节
##### 步骤1：打鸡蛋动画
1. 触发条件：进入模块后自动执行，无需用户操作；
2. 动画内容：鸡蛋从屏幕上方落下，磕在碗边，蛋液流入碗中，带轻微震动动效，总时长1500ms；
3. 动画结束后，自动进入选色环节，碗中显示液态淡奶油基底。

##### 步骤2：奶油选色&滴色素
1. 色卡展示：页面底部展示横向滚动色卡，色值包含：奶白、淡粉、蜜桃粉、淡芋紫、薰衣草紫、抹茶绿、薄荷绿、海盐蓝，无黄色系；
2. 选色交互：点击色卡，立即选中对应颜色；
3. 滴色素动画：选中颜色后，色素滴管从屏幕上方出现，滴下2滴对应颜色的色素液滴，液滴下落时长300ms，落入奶油后，有颜色扩散动画（500ms），奶油整体变为选中的颜色。

##### 步骤3：奶油打发交互
1. 视觉元素：碗中显示带颜色的液态奶油，右侧出现手持搅拌器图标；所有道具需为蜡笔手绘风，线条略带抖动感，不得使用工业图标风；
2. 交互逻辑：用户按住搅拌器，在奶油区域内做圆周拖动，每完成1个完整的圆周，奶油蓬松度增加25%，视觉上从液态稀状逐渐变为有纹理的蓬松固态；奶油纹理需呈现蜡笔涂抹后的颗粒感与不均匀抹面痕迹；
3. 完成条件：完成4个完整圆周后，奶油打发完成，弹出「奶油打发完成✅」提示，时长1500ms，自动消失；
4. 完成后，页面右下角出现「下一步」按钮，点击跳转至抹奶油&涂鸦模块。
#### 3.3 边界规则
- 未完成4个圆周打发，无法进入下一步；
- 搅拌器仅在奶油区域内可触发打发效果，区域外无效；
- 可随时切换奶油颜色，重新触发滴色素动画，重新打发。

### 模块4：抹奶油&涂鸦模块
#### 4.1 功能描述
用户在蛋糕胚上涂抹奶油、自由涂鸦，所有涂鸦内容将与蛋糕绑定，后续切割时同步切开。
#### 4.2 视觉&交互实现要求
1. 画布展示：页面居中展示选中的蛋糕胚，对应之前选择的单层/双层，尺寸占屏幕宽度的70%；
2. 操作规则：
   - 双层蛋糕流程：先处理底层蛋糕胚→完成抹奶油、涂鸦后，点击「叠加第二层蛋糕胚」按钮→再处理上层蛋糕胚；
   - 涂抹/涂鸦：用户按住鼠标/手指在蛋糕区域内拖动，即可绘制对应颜色的奶油涂鸦，画笔粗细可通过侧边的粗细调节条调整（10px-50px）；绘制笔触默认表现为蜡笔或奶油蜡笔混合质感，不使用钢笔、水彩、喷枪效果；
   - 辅助功能：侧边提供「撤销」「清空」按钮，支持撤销上一笔、清空所有涂鸦；
3. 图层规则：所有涂鸦内容存储在【用户涂鸦层】，使用蛋糕相对坐标，与奶油层、蛋糕胚层完全绑定，禁止使用屏幕绝对坐标；
4. 完成后，点击「下一步」按钮，跳转至蛋糕装饰模块。
#### 4.3 边界规则
- 仅在蛋糕胚轮廓内可绘制涂鸦，蛋糕外区域无法触发绘制；
- 双层蛋糕的底层和上层涂鸦分开存储，互不影响；
- 未涂抹任何奶油/涂鸦，也可进入下一步，不强制要求。

### 模块5：蛋糕装饰模块
#### 5.1 功能描述
用户自由选择装饰元素，拖拽摆放至蛋糕上，所有装饰元素与蛋糕绑定，后续切割时同步跟随蛋糕块。
#### 5.2 装饰元素分类（固定分类，不可修改）
| 分类 | 元素明细 |
|------|----------|
| 蜡烛类 | 细长蜡烛（白/粉/蓝/绿/金/银）、数字蜡烛（0-9）、迷你小蜡烛 |
| 食材类 | 草莓、蓝莓、青提、芒果片、猕猴桃片、树莓、白桃片、金色糖珠、银色糖珠、巧克力棒、爱心巧克力 |
| 插件类 | 小雏菊、茉莉花瓣、铃兰、爱心、星星、月亮、丝带蝴蝶结、Happy Birthday插牌、春尽夏生专属插牌 |
#### 5.3 交互实现要求
1. 装饰选择：页面底部展示分类tab，点击tab切换对应装饰列表，点击装饰元素，即可添加至蛋糕中心位置；装饰素材整体需呈现蜡笔贴纸感，边缘保留手绘描边与颗粒填色；
2. 拖拽操作：按住装饰元素，可自由拖拽至蛋糕任意位置，支持双指缩放、旋转（电脑端支持滚轮缩放）；
3. 删除操作：长按装饰元素2秒，弹出「删除」按钮，点击即可删除该元素；
4. 绑定规则：每个装饰元素的位置、缩放、旋转角度，均存储为相对于蛋糕中心的相对坐标，与蛋糕本体完全绑定；
5. 完成后，页面底部出现「完成装饰，去许愿」按钮，点击后先进入截图分享模块。
#### 5.4 边界规则
- 装饰元素可无限添加，无数量限制；
- 装饰元素可摆放至蛋糕外的页面区域，但切割时仅蛋糕轮廓内的元素会被处理；
- 蜡烛元素必须摆放，否则点击「完成装饰」时弹出提示「请至少添加一根蜡烛哦～」，无法进入下一步。

### 模块6：截图分享模块
#### 6.1 功能描述
用户选择模板，一键截图保存蛋糕成品，支持分享。
#### 6.2 实现细节
1. 触发条件：点击「完成装饰」按钮后，弹出截图分享弹窗，不跳转页面；
2. 模板选项：提供4套模板，点击即可预览效果：
   - 模板1：拍立得复古模板（白色拍立得边框，底部留白，边框允许加入轻微手绘描边与纸张质感）
   - 模板2：简约奶油风模板（圆角奶油边框，点缀小雏菊，并带蜡笔涂抹边缘）
   - 模板3：春尽夏生主题模板（暖奶黄边框，搭配主题书法字，并加入手账贴纸或蜡笔小涂鸦元素）
   - 模板4：魔法贺卡模板（带可交互遮盖层/拉手，用于在贺卡中揭示祝福文案）
3. 截图逻辑：点击「一键保存」按钮，仅截取**蛋糕本体+模板边框**，不包含页面按钮、工具栏等元素，生成高清 PNG（作为可编辑母本）；构建阶段可并行生成 WebP/AVIF 的线上交付版本。
4. 保存逻辑：电脑端触发图片下载，手机端触发保存至系统相册，同时弹出「保存成功」提示（1500ms）；
5. 魔法贺卡（新增功能）与流程衔接：
   - 在截图弹窗中提供“保存为魔法贺卡”选项，选择后系统将当前截图嵌入 `魔法贺卡模板` 并生成一张可交互的贺卡预览；贺卡支持在移动端/PC 端的拖动/滑动或点击揭示遮盖层的交互，揭示后显示隐藏的祝福文案（与封面副标题一致）。
   - 交互效果：贺卡揭示伴随纸张位移与轻柔翻页音效，揭示完成后弹出「已揭示祝福」确认提示；揭示动画长度建议 500-800ms。 
   - 解锁规则：系统将“在魔法贺卡中完成祝福揭示”作为解锁自由切割的条件（详见模块8），即：只有用户在贺卡上完成揭示动作后，才允许进入自由切割阶段，以避免后续切割与祝福展示语义冲突。 

#### 6.3 边界规则
- 不保存截图，也可点击「跳过，直接许愿」进入下一环节，不强制分享；
- 若用户选择“魔法贺卡”但未完成揭示交互，系统在切蛋糕阶段应阻止自由切割并引导用户回到贺卡完成揭示；
- 截图图片比例固定为3:4，适配社交平台分享；贺卡模板应在不同分辨率下保持揭示阈值的一致性（触控区域、滑动距离等）。

### 模块7：蜡烛仪式模块（点蜡→关灯→许愿→吹蜡）
#### 7.1 功能描述
完整实现生日许愿仪式，打造沉浸式氛围。
#### 7.2 分步骤开发细节
##### 步骤1：手动点蜡烛
1. 页面展示用户装饰完成的完整蛋糕，所有蜡烛默认熄灭状态；
2. 交互逻辑：用户手动逐个点击蜡烛芯，点击后蜡烛亮起暖黄色小火苗，火苗带持续晃动的帧动画，同时播放「叮」的轻柔音效；
3. 完成条件：所有蜡烛全部点亮后，自动进入关灯环节。

##### 步骤2：自动关灯&许愿提示
1. 关灯动画：最后一根蜡烛点亮后，页面整体亮度从100%线性降至20%，时长1000ms，仅保留蜡烛的烛光柔光，营造暗环境烛光氛围；
2. 许愿提示：关灯完成后，屏幕中央居中浮现淡奶油白色文字**「请许下你的心愿」**，文字带轻微呼吸闪烁效果，无强制倒计时，用户可自主停留。

##### 步骤3：吹灭蜡烛
1. 权限申请：进入该环节时，弹出麦克风权限申请提示，文案：「需要麦克风权限来模拟吹蜡烛哦，也可以选择手动点击熄灭蜡烛」；
2. 双操作模式：
   - 模式1（麦克风吹气）：麦克风检测到持续500ms以上、音量超过-30dB的吹气声，立即触发所有蜡烛同时熄灭的动画，火苗逐渐消失，时长800ms，搭配轻柔吹风声效；
   - 模式2（手动熄灭）：用户拒绝麦克风权限/权限申请失败，自动切换为手动模式，用户逐个点击蜡烛火焰，即可熄灭蜡烛，所有蜡烛熄灭后，流程继续；
3. 所有蜡烛熄灭后，页面亮度缓慢恢复至50%，自动进入切蛋糕互动模块。
#### 7.3 边界规则
- 未点亮所有蜡烛，无法进入关灯环节；
- 麦克风权限仅申请一次，拒绝后不再重复申请，默认使用手动模式；
- 蜡烛熄灭后，不可重新点亮，流程不可逆。

### 模块8：切蛋糕互动模块（核心重点，严格遵守）
#### 8.1 整体流程逻辑
分两个阶段，严格按顺序执行，不可提前解锁：
1. 第一阶段：首次切蛋糕（不可切割，仅触发提示）
2. 第二阶段：祝福展示完成后，解锁**自由切割功能**，支持用户自由切蛋糕，涂鸦、装饰同步切开

#### 8.2 第一阶段：首次切蛋糕（不可切割）
1. 触发条件：蜡烛全部熄灭后，页面左侧出现蛋糕刀图标，自动进入该阶段；蛋糕刀图标需为圆润蜡笔手绘风，不出现锋利写实刀具效果；
2. 交互逻辑：用户按住蛋糕刀，在蛋糕区域内拖动尝试切割，蛋糕仅产生轻微挤压形变动画，无任何切痕、无切割效果；
3. 提示规则：首次尝试切割后，屏幕中央弹出提示文字**「切不开哦，试着拉开蛋糕吧～」**，色值#8B5A2B，时长3000ms，自动消失；
4. 阶段结束：提示消失后，蛋糕进入可拉开状态，该阶段锁定切割功能，无论怎么操作都无法切开蛋糕。

#### 8.3 过渡说明：祝福揭示迁移到「魔法贺卡」
1. 设计调整说明：为避免在后续切蛋糕时出现语义或视觉冲突，原先的“在蛋糕上拉开揭示祝福”交互已迁移到 **模块6 的魔法贺卡**（截图分享后生成的可交互贺卡）。蛋糕本体不再承载拉开揭示交互，保留为切割与分享的真实交互对象。
2. 解锁逻辑：系统将“在魔法贺卡中完成祝福揭示”作为解锁自由切割的条件。用户若未在贺卡完成揭示而尝试切蛋糕，界面应阻止自由切割并弹出引导提示（例如「先去看看你的魔法贺卡，揭示祝福后即可切蛋糕」）。
3. 用户流程示例：
   - 用户在模块5 完成装饰并生成截图→在截图弹窗选择“保存为魔法贺卡”→在贺卡上执行拉开/滑动揭示操作→系统确认揭示完成并提示用户→用户返回切蛋糕模块，系统允许自由切割。
4. 交互与动画说明：贺卡揭示动画应平滑、具纸张质感（500-800ms），同时伴随轻柔翻页音效；切蛋糕模块保持原有的首次切不可切提示与自由切割解锁行为，但不再包含蛋糕本体的拉开祝福动画。

#### 8.4 第二阶段：自由切蛋糕（核心开发重点）
##### 8.4.1 核心功能要求
用户可自由绘制切割线，蛋糕沿切割线完全切开，同时【用户涂鸦层】【装饰元素层】【奶油层】【蛋糕胚层】同步沿切割线裁剪，实现涂鸦、装饰跟着蛋糕一起切开的效果，无错位、无脱节。

##### 8.4.2 交互实现细节
1. 蛋糕刀激活：蛋糕复位后，蛋糕刀图标高亮，提示「可以切蛋糕啦～」，自由切割功能解锁；
2. 切割线绘制逻辑：
   - 用户按住蛋糕刀，在蛋糕区域内拖动，即可绘制连续的黑色切割线；
   - 松开鼠标/手指后，系统判断切割线是否有效：**切割线两端必须完全超出蛋糕外轮廓，且线体连续无中断**，才会触发切割，否则切割线消失，不触发切割；
3. 切割动画：有效切割线确认后，蛋糕沿切割线缓慢分开，带轻微的奶油挤压动效，时长500ms，同时播放轻柔的奶油切割音效；
4. 图层同步裁剪规则（必须严格执行）：
   - 蛋糕四层复合画布，同时沿切割线进行路径裁剪，生成2个独立的蛋糕块画布；
   - 用户涂鸦层：沿切割线将连续的涂鸦路径断开，分别保留在对应蛋糕块的画布上，实现涂鸦被切开的效果；
   - 装饰元素层：判断装饰元素的重心位置，重心在哪个蛋糕块内，该装饰元素就跟随哪个蛋糕块移动；若装饰元素跨切割线且重心在切割线上，则沿切割线裁剪装饰元素；
5. 切割后交互：切开的蛋糕块，用户可按住自由拖动、分开摆放，拖动时，蛋糕块上的涂鸦、装饰同步跟随移动，无错位、无脱离；
6. 可重复切割：用户可对任意蛋糕块再次绘制切割线，进行二次切割，规则同上。

##### 8.4.3 边界规则
1. 最多支持切割成8块，超过8块后，弹出提示「已经切得足够碎啦，好好享用吧～」，无法继续切割；
2. 仅蛋糕轮廓内的区域可绘制切割线，蛋糕外区域无效；
3. 切割线必须连续，中断、未超出蛋糕轮廓的线均为无效线，不触发切割；
4. 蛋糕块拖动不可超出页面边界，不可重叠后合并。

### 模块9：收尾&重置模块
#### 9.1 功能描述
流程收尾，展示主题结语，支持重新制作蛋糕。
#### 9.2 实现细节
1. 触发条件：用户完成蛋糕切割后，停留3秒，页面自动缓缓亮起，亮度恢复至100%；
2. 结语展示：屏幕中央居中展示结语文案，字体为圆润奶油体，色值#8B5A2B，带轻微淡入动画：
   **春尽夏生，岁岁欢愉，愿你万事顺意，平安喜乐**
3. 重置按钮：结语展示3秒后，页面右下角出现「重新制作蛋糕」按钮，点击后，页面清空所有用户数据，跳转回首页封面，流程重启；

#### 9.3 边界规则
- 结语展示不强制，用户可随时点击重置按钮，无需等待；
- 重新制作后，所有之前的涂鸦、装饰、切割数据全部清空，不可恢复。

## 四、非功能需求
### 4.1 性能需求
1. 页面首屏加载时间≤2秒，所有动画、交互响应时间≤100ms，无卡顿、无掉帧；
2. 图片资源全部压缩，单张图片大小≤200KB，总资源体积≤5MB；
3. 低配置手机端操作无卡顿，Canvas绘制无延迟。
### 4.2 兼容性需求
1. 浏览器兼容：Chrome 90+、Safari 14+、Edge 90+、微信内置浏览器、手机端主流浏览器；
2. 系统兼容：iOS 13+、Android 10+、Windows 10+、macOS 11+；
3. 触屏操作与鼠标操作完全兼容，无操作盲区。
### 4.3 稳定性需求
1. 全程无崩溃、无白屏、无报错，流程可完整闭环执行10次以上无异常；
2. 刷新页面后，自动回到首页封面，无数据残留、无状态错乱；
3. 权限申请失败、网络异常等情况，有降级方案，不影响流程继续。
### 4.4 易用性需求
1. 无需注册登录、无需授权（除麦克风外），打开网站即可使用；
2. 所有操作有明确引导，无需教程即可上手，无复杂操作；
3. 全程无广告、无弹窗、无付费环节，纯免费体验。

## 五、资源规范
1. 图片资源（源文件与线上交付分离）：
   - 源文件（master）：所有装饰元素、蛋糕胚、图标、相框、纸张纹理、蜡笔笔触素材请保留为 PNG（透明底）且为 2x 分辨率，作为可编辑的母本用于后续修图、训练 LoRA/DreamBooth、inpainting/mask 流程使用。
   - 线上交付：构建/发布阶段请将 PNG 批量转为 WebP 或 AVIF（支持 alpha 的无损/near-lossless 模式）以减小网络体积并提高加载性能；同时在页面中提供 PNG 回退以兼容极少数不支持 WebP 的旧平台。
   - 前端使用建议：在页面中采用 `picture`/`srcset` 提供 WebP 优先、PNG 回退的策略；纹理类素材建议使用无损或 near-lossless 导出以保留蜡笔颗粒感。
2. 音效资源：所有音效为无版权纯音乐、轻柔音效，格式MP3，单文件大小≤500KB；
3. 字体资源：书法字体、圆体均为可免费商用字体，禁止使用有版权风险的字体；字体气质需与蜡笔绘本感协调，避免科技感、商务感字体；
4. 风格资源：建议额外准备纸张底纹、蜡笔笔触纹理、手绘边框、贴纸、涂鸦角标等全局装饰素材，用于统一首页、弹窗、分享模板、按钮与提示组件的风格。
